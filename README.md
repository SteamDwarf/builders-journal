# Журнал учета строительных работ (WorkLogs)

Веб-приложение для мониторинга, создания и редактирования записей о выполненных строительных работах на объектах. Позволяет вести учет объемов, типов работ и контролировать исполнителей (бригадиров). Весь проект полностью контейнеризирован и готов к локальному развертыванию через Docker Compose.

---

## 🛠 Стек технологий и обоснование

Архитектура проекта выполнена по методологии **Feature-Sliced Design (FSD)**, что обеспечивает высокую масштабируемость, контролируемые связи между модулями и легкий онбординг новых разработчиков.

### Frontend

- **React 18+ & TypeScript** — промышленный стандарт для построения надежных интерфейсов со строгой типизацией, минимизирующей ошибки на этапе компиляции.
- **Ant Design 5 (AntD)** — готовая библиотека UI-компонентов с мощной экосистемой (модальные окна, таблицы с ленивой фильтрацией и адаптивная сетка), поддерживающая динамическую смену тем через Design Tokens.
- **@tanstack/react-query (v5)** — продвинутый стейт-менеджер для серверного состояния. Берет на себя кэширование, автоматическую инвалидацию кэша при мутациях (CRUD-операциях) и оптимистичные обновления.
- **Axios** — надежный HTTP-клиент с настроенными интерцепторами для обработки базовых URL и централизованной обработки ошибок.

### Backend & DB

- **NestJS** — прогрессивный Node.js фреймворк с модульной архитектурой на базе TypeScript, обеспечивающий соблюдение принципов SOLID из коробки.
- **Prisma ORM** — современный инструмент для работы с базами данных со стопроцентной автогенерацией типов на основе схемы данных.
- **MySQL 8.0** — надежная реляционная СУБД для обеспечения целостности связанных данных (журнал работ ↔ справочник видов работ).

## 🚀 Пошаговая инструкция по запуску через Docker

Всё окружение (База данных, Бэкенд и Фронтенд) поднимается в изолированных контейнерах. Понадобятся только установленные **Docker** и **Docker Compose**.

### 1. Настройка переменных окружения (.env)
В корневой директории проекта создай файл .env:

Фрагмент кода

```
BACKEND_PORT=5000
FRONTEND_PORT=5173

MYSQL_ROOT_PASSWORD=root_secure_pass
MYSQL_DATABASE=worklogs_db
MYSQL_USER=worklog_user
MYSQL_PASSWORD=user_secure_pass
```

### 2. Запуск

Выполни в корне проекта:

```Bash
docker compose up --build
```

### 3. Ссылки для доступа

Frontend: http://localhost:3000
Backend API: http://localhost:8000/api
MySQL: localhost:3306
